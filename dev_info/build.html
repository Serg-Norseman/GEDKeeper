How to build GEDKeeper

Microsoft Windows

Method 1

Run the executable build_all_mswin.x86.cmd from the command line.

Method 2

Having the following toolchains is a prerequisite for building GEDKeeper on a Windows host:

  • MSYS2/MINGW shell.
    It MAYBE is enough to use the Windows 10's (Ubuntu's) BASH shell. I didn't test that.
  • GNU make package installed in the MSYLS2/MINGW shell.

To begin, change to the directory where the GEDKeeper project is located:
$ cd /d/vsp/gedkeeper

Then simply run the make utility:
$ make

The default target ("all") builds all binaries (GEDKeeper itself and all available plug-ins).

All subprojects also have their makefiles. However, those makefiles' default target ("all") only builds a release and doesn't install it. Use the "install" option to install a specific project's binaries.

The following variables are supported by all make files:

  • hardwareplatform
    Target hardware platform. Possible values are "any", "x86-64" and "x86". "any" is the default.
  • softwareplatform
    Target software implementation on the hardware platform. Possible values are "windows" and "linux". "windows" is the default.
  • releasetype
    Release type. Possible values are "debug" and "release". "release" is the default.
  • dotnet
    Determines the .NET framework to use. Possible values are "dotnet20" and "dotnet46". "dotnet20" is the default.

Therefore, the following calls are equivalent:
$ make
$ make hardwareplatform=any softwareplatform=windows releasetype=release dotnet=dotnet20

The currently supported targets are:

  • all
    Builds a project and all its dependencies. On the root makefile it also installs GEDKeeper and all available plug-ins.
  • install
    Available only for a specific project. This goal builds, if necessary, the project and its dependencies and then installs all binaries into the GEDKeeper's directory.
  • clean
    Remove all files made by "all" or "install" target.
