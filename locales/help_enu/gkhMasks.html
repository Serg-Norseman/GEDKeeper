Filter Masks

You use masks in GEDKeeper to narrow down persons, events, or locations. The text of a mask consists of general printable characters allowed for object names, and two special symbols '*' and '?' for wildcard support.

The term mask is used because the '*' and '?' symbols mask some arbitrary parts of a value.

  • '*' inside a pattern matches any number of any characters.
  • '?' matches one single character.

If you want to specify several different masks, you can use the "|" symbol to delimit each mask. For example, to get all persons with names Mitch and John you can use the "*Mitch*|*John*" mask. The "|" symbol acts as an "OR" conjunction, therefore any person whose name matches one of the masks, matches the entire "Mitch*|John*" mask.

Any person's name, event, or location will always match the '*' mask. When you use such a mask as a filter, it will show all objects.

When you need to see all persons with "Ivanov" as beginning part of the last name, you would use a "Ivanov*" mask in the "Name mask" filter. The asterick symbol ("*") matches any characters after the initial substring "Ivanov". This way you get all persons with Ivanov, Ivanovskih, and so on as a last name. If you want to filter out all persons with "Vasiliy" as a first name, you should use a "*Vasiliy *" mask; thus you are masking any last name and surname. To get all women with Vasilevna surname you use "*Vasilevna", masking any first name and last name.

Examples:

  • Names "Litvin Ivan Andreevich" and "Semenov Ivan Andreevich" can be filtered with a "*Ivan Andreevich" mask.
  • Any person with first name, last name, or surname having "Ivan" inside, matches the "*Ivan*" mask.
  • Any person with the name "Natalia" or "Natalya" is matched with the "Natal?a*" mask.
