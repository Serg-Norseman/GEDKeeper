File Encryption

GEDKeeper supports storing and loading encrypted GEDCOM files. The GEDCOM standard does not support encryption, therefore GEDKeeper chose its own implementation for file encryption based on industry standards.

Currently GEDKeeper uses an AES algorithm for encryption and SHA1 for signing.

When you are saving a database you must choose "GEDKeeper encrypted GEDCOM files (*.geds)" as the target file format. Files of this format have a file extension of ".geds". After you have selected the file format and entered the file name, you are prompted to enter a password. When you are about to load an encrypted file, you select the same file format and enter your password when you are prompted.

Attention: GEDKeeper uses a cryptographically secure encryption method and following "security best practices" it does not store the password within the encrypted files. Therefore, if the password is lost or forgotten nobody, including the GEDKeeper author, will be able to open your database file. Be prudent and enter your password carefully. Any symbol inadvertently added can be a cause for losing your file. Please note that passwords are case sensitive.

Future releases will likely include:

  • Allowing the user to choose the encryption algorithm.
  • Prompting for the password only once for the same file.
  • Adding a "password repeat" field when a file is saved the first time (to help avoid typos in the password).
