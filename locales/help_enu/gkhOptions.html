User interface

Highlight unparented

Highlighting a person with no parents or family improves navigation through the list. The following color scheme is applied: a white background is used for persons with family and parents, a flaxen background for persons without a family, and light red for persons without parents. Colors do not have an effect on the information.

Trees

There are two setting groups for family tree generation: how persons are represented in the tree and how the tree appears when it is rendered (its' decor).

Person representation in the tree

Here you define what data you will see in the tree: last name, first name and surname, birthday and obit, part of dates, etc.

When "Relation degree" option is on, every person has visible relation degree, that is calculated on each tree rebuild relative to the central person in the tree.

"Additional symbols" shows special symbols for some categories of people. There are special icons for veterans of the World War II, died, veterans of Labor, cavalier of Cross of St. George, repressed. These categories are defined in the user-defined reference section in person editor.

"Embellished appearance" controls some additional graphical elements and shadow.

"Hide passed away in childhood" option becomes helpful when you handle several sources strongly, because it can reduce the number of visible persons and make your task much simpler.

Appearance (color and font)

This setting group defines colors and the main font used in tree rendering.

Pedigrees

Here you set up pedigree generation.

  • "Include attributes of individuals" controls if a person's life events will be included in the pedigree.
  • "Include notes" option states that text notes from a person's biography will be included in the pedigree.
  • "Include sources" option checks if pedigree includes source's number that references a person. There is the source list at the bottom of pedigree.
  • "Format" option has two possible values: "Excess" and "Compact". These control the level of details for a person description printed on the pedigree. The compact format is generally used since the excess version may contain too much unnecessary information.
