Filtering Person Records

You use filtering to restrict the number of persons you see at one time. The database size grows with each record, so one day you may find it difficult to find the person(s) you are looking for. This is when filters become extremely useful.

To set up the filters you use the toolbar button, the "Edit\Filter" menu, or the hotkey [Ctrl + F].

There are two types of filters: standard and additional specific filters. Standard filters are used for any type of record in the database; it is available on the "Fields filter" tab in the filtering dialog. Additional specific filters are used for person records only, and are available on the "Specific filter" tab when filtering an open person list.

Fields filter works identically for all person records. A user selects a target field (a column in the database's list) on which the filter will be applied, then add a filter condition and value. Several selection criteria may be applied as required.

For example, you want to select all people with an Ivanov-like last name who lived in Samara. To do this, open the filtering dialog, go to the filter's first row, and select "Full name" value in the "Field" dropdown list. Set the condition to "Contains", and value to "Ivanov". Then go to the next filter's row and set field to "Residence", condition to "Contains", and value to "Samara".

Specific filtration is used to improve performance and work efficiently while handling person records. Such a filter has the following selection criterions: sex, is a person alive, patriarch's bookmarks, all name parts, residence, events, groups, and sources. There are masks available when one is filtering by name, life events, or residence.

When processing person records, a user can use simple filters along with additional specific filters.
